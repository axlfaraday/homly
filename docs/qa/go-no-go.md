# Go / No-Go checklist

## Bloqueantes (No-Go)
- Bugs severidad critica en pagos, reserva, auth o autorizacion.
- Webhooks de pago inconsistentes sin reconciliacion.
- Flujos criticos sin cobertura e2e minima.
- Fallos A11y severos en navegacion teclado-only.

## Requerido para Go
- 0 bugs bloqueantes.
- P95 bajo objetivo acordado en endpoints criticos.
- Landings SEO indexables sin errores criticos.
- Runbooks operativos de incidentes y reembolsos.
